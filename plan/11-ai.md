# 11 — AI-Native Features

Blume treats LLMs as first-class consumers of docs (resolved 09-T). Most of this is
**static and zero-infra**; only interactive "Ask AI" needs compute, and even that
stays lock-in-free via **bring-your-own provider/key** (consistent with the "no
hosted control plane" non-goal in [00-vision.md](./00-vision.md)).

## Static, zero-infra (on by default)

These ship in the build output and work on any static host:

### `llms.txt` + `llms-full.txt`
Generated at build per the [llmstxt.org](https://llmstxt.org) convention:
- **`/llms.txt`** — a curated map of the site: title, description, and a structured
  list of pages (grouped by nav) with links and summaries.
- **`/llms-full.txt`** — the entire docs corpus concatenated as clean markdown for
  direct ingestion.

### Per-page raw markdown
Every route is also retrievable as clean markdown — e.g. `/<route>.md` (and/or
content negotiation). Lets an LLM (or human) grab the exact source of any page
without scraping rendered HTML.

### Copy as Markdown
A per-page action ("Copy page as Markdown") in the page header/menu — copies the
clean markdown of the current page to the clipboard.

### Open in ChatGPT / Claude
A per-page menu that deep-links the current page into ChatGPT or Claude as a
prompt (e.g. "Read <page-url>.md and help me with…"), so readers can jump from a
doc straight into a model with the page as context.

## Interactive "Ask AI" (BYO provider)

An in-docs assistant that answers questions grounded in the docs corpus:

- **Entry point:** integrated into the ⌘K command palette (an "Ask AI" mode) and/or
  a dedicated panel. Reuses the `SearchButton`/search UX from
  [06-navigation.md](./06-navigation.md).
- **Built on the AI SDK** (Vercel AI SDK) — streaming responses, provider-agnostic
  (Anthropic, OpenAI, Google, …). Default to the latest capable models; configurable.
- **Retrieval:** grounded on the same indexed content used for search (and/or
  `llms-full.txt`), with cited sources linking back to pages.
- **Bring-your-own key / provider:** Blume ships the feature, not the inference.
  Teams supply a provider + key. No Blume-hosted service, no lock-in.

### Server-only by design (resolved 09-Z)
Ask AI is **always server-proxied** — the provider key lives on a server route and
is **never exposed to the browser**. There is no client BYO-key-in-browser mode
(that would leak the key on any public site).

Consequences:
- Enabling `askAI` requires a **server runtime**, so it flips `output` to `server`
  (resolved 09-B2). A small serverless/Node route proxies the AI SDK call.
- A **fully static** deployment can still ship everything else — llms.txt, raw
  `.md`, copy-as-markdown, open-in-LLM. Only interactive Ask AI needs the server.
- The route enforces server-side guards (rate limiting, allowed origins) so the
  key can't be abused.

## Configuration

```ts
// blume.config.ts (excerpt)
ai: {
  llmsTxt: true,            // /llms.txt + /llms-full.txt        (static)
  rawMarkdown: true,        // per-page .md endpoints            (static)
  copyMarkdown: true,       // "Copy as Markdown" action         (static)
  openInLLM: ["chatgpt", "claude"],   // per-page deep links     (static)
  askAI: {
    enabled: true,
    provider: "anthropic",  // AI SDK provider (key read server-side from env)
    model: "claude-...",    // default to latest capable model
    // server-only: enabling this requires `output: "server"` (09-Z)
  },
},
```

All four static features default on; `askAI` is opt-in, **server-only**, and reads
its key from server env — never from config or the client.

## Components & overrides

AI surfaces are normal Blume components, overridable via `components.tsx`
([05](./05-customization.md)) and themeable like everything else:
- `layout.AskAI` — the assistant panel/dialog.
- `layout.PageActions` — the per-page menu hosting Copy-as-Markdown / Open-in-LLM.

## Roadmap placement

- **Static AI** (llms.txt, raw `.md`, copy, open-in-LLM) — lands with search/build
  (around M6–M7); cheap and high-leverage.
- **Ask AI** — follows once content indexing is stable; **server-proxied** route +
  the `AskAI` panel, grounded with cited sources (09-Z).
- **Post-1.0:** expose docs as an **MCP server** so agents can query a Blume site
  natively; richer retrieval/eval tooling.

See [08-roadmap.md](./08-roadmap.md).
